Vamos estruturar esse aplicativo web dividindo a modelagem do banco de dados (a Tabela SUSEP), a interface (Frontend) e a lógica de cálculo (Backend), garantindo que o algoritmo respeite as regras de indenização por Invalidez Permanente por Acidente (IPA). Aqui está o passo a passo de como construir essa *feature*: \#\#\# Passo 1: A Base de Dados (JSON da Tabela SUSEP) A Tabela SUSEP possui percentuais fixos para a perda total de cada membro ou órgão 1-3. Precisamos mapear isso em um objeto JSON que será consumido pelo frontend. javascript const tabelaSusep \= { // Danos Totais (100%) "visao\_ambos\_olhos": 100, "ambos\_membros\_superiores": 100, "ambos\_membros\_inferiores": 100, "alienacao\_mental": 100, // Membros Superiores "braco\_direito": 70, "braco\_esquerdo": 70, "mao\_direita": 60, "mao\_esquerda": 60, "polegar": 25, "indicador": 15, // Membros Inferiores "perna\_direita": 70, "perna\_esquerda": 70, "pe\_direito": 50, "pe\_esquerdo": 50, // Diversos "visao\_um\_olho": 30, "surdez\_ambos\_ouvidos": 40, "surdez\_um\_ouvido": 20 // ... continue mapeando todos os itens da tabela legal \[1, 4-9\] };  \#\#\# Passo 2: A Interface (UI) \- O Boneco Lego e os Inputs A interface será dividida em duas áreas: **Visualização do Corpo** e **Painel Financeiro**. **1\. O Boneco Interativo (SVG):** \* Use um gráfico vetorial (SVG) de um boneco com articulações bem definidas (estilo Lego ou *low-poly*). \* Cada segmento do corpo (cabeça, ombro, braço, antebraço, mão, coxa, perna, pé) será uma tag  ou  com um id correspondente à chave do nosso JSON (ex: id="mao\_direita"). \* Adicione um evento de *onClick*. Quando o usuário clica num segmento (que muda de cor via CSS :hover), abre-se um modal. **2\. O Modal de Avaliação do Segmento:** Neste modal, o perito/usuário informa o **Grau de Redução Funcional** daquele membro específico. A regra da SUSEP diz que, se a perda não for total, aplica-se um redutor sobre o percentual da tabela 10, 11\. \* *Radio buttons* para o grau de perda: \* Perda Total (100%) \* Perda Parcial Máxima (75%) 10, 11 \* Perda Parcial Média (50%) 10, 11 \* Perda Parcial Mínima (25%) 10, 11 **3\. O Painel Financeiro (Inputs):** Crie um formulário simples para o usuário inserir a base de cálculo financeira: \* input type="number" id="valorBase" \-\> Aqui o usuário digita o **Valor Segurado** (ex: R$ 50.000 de uma apólice privada) ou o **Valor do DPVAT** (historicamente fixado em até R$ 13.500 para invalidez). \#\#\# Passo 3: O Algoritmo de Cálculo (Regra de Negócio) No seu *service* em JavaScript, você vai iterar sobre os segmentos corporais que o usuário clicou e avaliou, multiplicando o percentual da tabela SUSEP pelo grau de perda funcional, e depois aplicar isso ao valor financeiro 10, 11\. **Atenção às regras de trava da SUSEP:** 1\. A soma total de todas as lesões não pode ultrapassar 100% 12\. 2\.
